## 4. [강의] 생성형 AI 동향 이해, HRD 적용 관점 정리

### 4-1. 학습목표

- 생성형 AI의 발전 흐름을 이해한다.
- LLM, Transformer, Attention의 핵심 개념을 HRD 관점으로 단순화해 설명할 수 있다.
- HRD 업무 전 과정에서 AI 적용 가능 지점을 구체적으로 식별한다.

### 4-2. 핵심 내용

- 생성형 AI 발달 역사 요약
  - 2017년: 구글의 'Transformer' 아키텍처 발표 (방대한 데이터의 문맥을 병렬로 파악)
  - 2020년: GPT-3 등 거대언어모델(LLM)의 파라미터 규모 경쟁 본격화
  - 2022년 말: ChatGPT 공개를 기점으로 대화형 생성형 AI의 대중화 및 실무 도입 가속화
  - 2024년 이후: 단순 언어모델을 넘어 복잡한 문제해결을 위한 추론(Reasoning) 특화 모델 고도화
- 현재 기준(2026-04) 최상위(SOTA, State-of-the-Art) 모델 동향
  - 프론티어(Frontier) 모델: OpenAI GPT-4.5 / GPT-5(예정) 및 o1·o3(추론 특화), Google Gemini 1.5 / 2.0 Pro, Anthropic Claude 3.5 Sonnet / Opus
  - 오픈소스 및 소형화(SLM): Meta Llama 3 시리즈, Mistral 등 특정 기기(온디바이스) 및 도메인 특화 모델의 확대
- 최근 생성형 AI의 진화 흐름
  - 텍스트 중심 LLM에서 이미지·음성·문서를 융합하는 멀티모달 AI로 확장
  - 단순히 묻고 답하는 것을 넘어 스스로 계획하고 행동하는 Agentic AI로 확장
  - 소프트웨어 자동화를 넘어 물리적 로보틱스와 결합한 Physical AI로 진화
- 대표 서비스 예시(2026-04 기준, 예시는 빠르게 바뀔 수 있음)
  - 범용 생성형 AI: ChatGPT, Gemini, Claude
  - AI 코딩/자동화 도구: Codex, Gemini CLI, Claude Code, GitHub Copilot
- 생성형 AI의 작동 원리
  - LLM은 대규모 텍스트 패턴을 바탕으로 다음 토큰을 예측하는 모델
  - Transformer는 입력 간 관계를 병렬적으로 파악하는 구조
  - Attention은 문맥 속에서 어떤 정보가 더 중요한지 가중치를 두는 메커니즘
  - 멀티모달 모델은 텍스트, 이미지, 음성, 영상, 문서를 함께 이해하고 생성
- HRD와 AI의 접점
  - 분석: 요구분석, 교육대상자 분류, VOC 정리
  - 설계: 과정안 작성, 메시지 구조화, 학습경험 설계
  - 개발: 교안, 슬라이드, FAQ, 웹페이지, 메일, 보고서 초안 생성
  - 운영: 문의응대, 일정공지, 후속안내, 실습자료 배포
  - 평가: 만족도 분석, 정성의견 요약, 토픽 도출, 경영진 보고자료 작성
- 프롬프트 엔지니어링 최소 구조
  - 역할: 누구의 관점에서 답할지
  - 맥락: 어떤 배경과 제약이 있는지
  - 입력자료: 표, 문서, 설문, 사례 등
  - 출력형식: 표, 메일, 보고서, FAQ, HTML 등
  - 품질기준: 정확성, 톤앤매너, 길이, 금지사항
- 생산성 팁
  - STT 활용: Windows 음성입력 `Win + H`
  - 반복 업무는 템플릿화하고, 결과물은 사람이 마지막 판단을 담당

## 5. [실습 1] 교육운영 효율화를 위한 AI 챗봇/웹페이지 만들기

### 5-1. 실습 목표

- 교육민원과 반복 문의를 줄이기 위한 정보 제공 방식을 설계한다.
- 챗봇과 정적 웹페이지 중 어떤 방식이 더 적합한지 판단한다.
- GitHub Pages를 활용해 간단한 교육 운영용 페이지를 배포한다.

### 5-2. 핵심 메시지

- 정보가 자주 바뀌지 않고 정확성이 중요하면 FAQ 웹페이지가 유리하다.
- 개인화된 탐색이나 대화형 응답이 중요하면 GPTs 또는 사내 챗봇이 유리하다.
- 현실적으로는 "웹페이지 + 필요 시 챗봇"의 조합이 운영 리스크가 가장 낮다.

### 5-3. 실습 결과물 정의

- 산출물: chatGPT GPTs, 교육 안내용 마이크로사이트 1종
- 참고 데이터: `Webpage_Sample_Data.md` (교육 과정 안내 웹페이지 제작에 필요한 가상 정보 일체)
- 필수 섹션:
  - 과정 개요
  - 일정/장소
  - 준비물 및 사전과제
  - FAQ
  - 문의처
  - 자료 다운로드 또는 링크
  - 모바일 접속용 QR 코드

### 5-4. 실습 절차(수정 제안)

1. GitHub에서 새 저장소를 생성한다.
2. 로컬에 clone을 한다
3. 웹페이지를 만드는데 필요한 정보들을 정리한다
   - 교육 목적
   - 타깃 사용자
   - 반드시 포함할 정보
   - 모바일 우선 여부
   - 톤앤매너
4. 웹페이지 형태로 만들어 달라고 한다.
5. AI 도구가 만든 초안을 검토하고 다음 항목을 사람이 수정한다.
   - 오탈자
   - 실제 일정/장소/문의처
   - 사내 표현 방식
   - 개인정보 노출 여부
6. 수정된 내용을 GitHub에 push한다
7. GitHub Pages로 배포 후 모바일 화면에서 최종 확인한다.
8. QR 코드를 생성해 현장 배포자료에 삽입한다.

### 5-5. 실습 시 사용할 프롬프트 예시

```text
HRD 과정 안내용 단일 페이지 웹사이트를 만들어줘.

목표:
- 교육 참가자가 일정, 장소, 준비물, FAQ, 문의처를 빠르게 확인할 수 있게 하기
- 모바일에서 먼저 읽히도록 설계하기
- 대상자명단에서는 전화걸기, 문자보내기 버튼이 있어야함

반드시 포함할 섹션:
- 과정 개요
- 일정/장소
- 준비물
- FAQ 6개
- 문의처
- 자료 다운로드 버튼

디자인 조건:
- 제조업 대기업의 신뢰감 있는 톤
- 과도한 애니메이션 금지
- 가독성 우선
- HTML/CSS/JavaScript를 분리해서 작성
```

## 6. [시연 1] 교육 콘텐츠 및 피드백 자동화

주제: 리더십 교육 F/U 개인화 메시지 실습

### 6-1. 시연 목표

- 교육 수강자의 Action Plan 데이터를 활용해 개인화된 후속 메시지 초안을 자동 생성한다.
- 메일 작성 시간을 줄이면서도 개인별 맥락을 유지하는 방법을 보여준다.
- 개인화된 메일을 발송하는 Tool을 활용한다.

### 6-2. 실습 데이터 구조 (`교육_액션플랜_데이터.csv`)

본 시연에서는 50명 규모의 가상 수강자 데이터를 활용한다. 시연에 사용될 데이터의 주요 컬럼은 다음과 같다.
- **기본 정보**: `name`(이름), `company`(소속사), `department`(부서), `position`(직책)
- **과정 산출물**: `aspiration`(학습 포부), `action_plan`(핵심 실행계획), `expected_behavior`(기대 행동)
- **피드백 정보**: `training_comment`(수강 코멘트), `top_takeaway`(가장 인상 깊었던 점)
- **운영 정보**: `email`(이메일), `manager_name`(담당 매니저)

### 6-3. 권장 데이터 흐름
1. 스프레드시트 데이터(`교육_액션플랜_데이터.csv`) 로드 및 전처리
2. LLM API에 전달할 프롬프트 템플릿 설계 (이름, 소속, 실행계획, 코멘트, 매니저명 등 변수 매핑)
3. 지정된 담당 매니저(`manager_name`) 명의의 개인별 후속지원 메시지 초안 자동 생성
4. HTML 메일 본문 형식으로 변환
5. 발송 전 어조, 민감표현, 인사말 등 사실관계 최종 검수
6. 이메일 발송 Tool(`FILE_엑셀에서메일발송_XLMultimail_v2.6.0_20240117.zip`)을 활용하여 전체 일괄 발송

### 6-4. 시연에 사용되는 도구 및 파일

| 구분 | 파일명 | 설명 |
|---|---|---|
| 자동화 스크립트 | `SCRIPT_glm_feedback_automation.py` | CSV 데이터를 읽어 LLM API로 개인별 피드백 초안을 생성하고, HTML 메일 미리보기와 대시보드까지 출력하는 Python 스크립트 |
| 실습 데이터 | `sample_data/교육_액션플랜_데이터.csv` | 50명 규모 가상 수강자의 액션플랜·피드백 데이터 |
| 메일 발송 도구 | `FILE_엑셀에서메일발송_XLMultimail_v2.6.0_20240117.zip` | Excel 기반 일괄 메일 발송 애드인 (XLMultimail) |

## 7. [실습 2] 교육 데이터의 AI 기반 자동화 전략

주제: 수집-전처리-분석-결과보고서 시각화까지

### 7-1. 실습 목표

- 교육 만족도 데이터를 정량/정성으로 나누어 분석하는 흐름을 이해한다.
- 수작업 보고를 줄이고, 반복 가능한 분석 파이프라인을 설계한다.
- 최종 분석 결과물이 담긴 데이터 대시보드를 단일 HTML 보고서로 연계하여 생성한다.

### 7-2. 다룰 데이터 (`교육만족도_설문_데이터.csv`)

- **1차 데이터**: 300명 규모의 과정 만족도 조사 결과 (`교육만족도_설문_데이터.csv`)
- **정량 데이터**: 각 문항 특성에 맞추어 통계 수치 도출 (강사만족도, 내용적합성 등은 리커트 척도 기반 평균(Average)을 구하고, 추천의향 문항은 NPS(Net Promoter Score)를 별도로 계산)
- **정성 데이터**: 주관식 수강소감에 대해 한국어 특화 감성분석(KOTE; Korean Online Text Emotion) 및 핵심 토픽모델링을 수행하여 대표 인사이트 도출
- **2차 산출물**: NPS 차트, 감성분포, 토픽모델링 워드클라우드 등이 결합된 HTML 기반 최종 요약 보고서

### 7-3. 권장 분석 흐름

| 단계 | 입력 | 처리 | 출력 |
|---|---|---|---|
| 수집 | 설문 원본 | 결측치 및 항목 속성 점검 (`교육만족도_설문_데이터.csv`) | 분석 대상 데이터셋 |
| 전처리 | 정량/정성 응답 | KOTE 감성분석을 위한 형태소/텍스트 정제, 비식별화 | 정제된 분석용 데이터 |
| 정량분석 | 점수형 응답 | 문항 특성별 분석 로직 적용 (만족도 문항군 평균 도출, 추천도 기반 NPS 도출) | 주요 통계 요약 및 차트 생성용 값 |
| 정성분석 | 서술형 응답 ('수강소감') | KOTE 모델 기반 긍부정/세부 감성분류, 토픽모델링(LDA 등)으로 주요 키워드 추출 | 텍스트 분석 핵심 인사이트 |
https://github.com/searle-j/KOTE 레포지토리에서 모델다운로드 및 활용가이드 확인
https://arxiv.org/pdf/2403.16248 LLM기반의 토픽모델링
| HTML 생성 | 정량/정성 결과 종합 | 정량 수치와 정성 토픽 분석 결과를 하나의 문서로 결합 (시각화 포함) | 경영진 보고용 HTML 통합 보고서 |
| 환류 | 최종 요약 보고서 | 데이터에 나타난 강약점을 기초로 다음 차수 액션 도출 | 교육 개선 실행 과제 |

### 9-3. 수업 말미에 추천할 Utility

- `PowerToys`
  - Windows 생산성을 높여주는 무료 유틸리티 모음으로, 실습 중 창 정리와 빠른 실행, 텍스트 추출에 특히 유용하다.
  - 주요 기능:
    - `FancyZones`: 여러 창을 미리 정한 레이아웃에 맞춰 배치할 수 있어, 강의자료, AI 도구, 메모 창을 동시에 띄워도 화면이 쉽게 흐트러지지 않는다.
    - `Text Extractor`: 이미지나 영상, 웹페이지 화면에 보이는 텍스트를 OCR로 바로 복사할 수 있어, 예시 화면의 문구를 실습 자료로 재활용할 때 편하다.
    - `PowerToys Run` 또는 `Command Palette`: 앱, 파일, 폴더, 명령을 빠르게 실행할 수 있어 실습 도중 필요한 도구 전환 속도를 높여준다.
    - `Always On Top`: 참고해야 할 창을 항상 위에 고정할 수 있어 체크리스트나 공지사항을 띄워두고 진행하기 좋다.
- `ShareX`
  - 화면 캡처, 스크롤 캡처, GIF 녹화, OCR, 주석 편집까지 한 번에 처리할 수 있어 교육 운영 자료 제작에 매우 실용적이다.
  - 주요 기능:
    - `Region Capture / Scrolling Capture`: 긴 웹페이지나 설문 결과 화면 전체를 한 장으로 캡처할 수 있어 FAQ 페이지, 대시보드, 결과 보고 화면을 공유하기 좋다.
    - `Annotation / Image Editor`: 화살표, 박스, 흐림, 하이라이트 등을 넣어 매뉴얼이나 안내 이미지를 빠르게 만들 수 있다.
    - `Screen Recording / GIF`: 짧은 사용법 시연을 GIF나 영상으로 남길 수 있어, 교육 전후 안내자료 제작에 적합하다.
    - `OCR / Workflow`: 캡처한 이미지에서 텍스트를 뽑거나, 캡처 후 저장·복사·업로드 같은 후속 작업을 자동화할 수 있다.
